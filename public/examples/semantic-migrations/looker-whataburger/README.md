# Whataburger-style Looker benchmark

This simulated LookML project follows Google's documented project file conventions: one model file, one view file containing six views, and one version-controlled dashboard file. It is reconstructed from OmniKit's Whataburger-style demo and is not a production Looker export or brand-owned dataset.

Use it to verify deterministic LookML parsing and the reviewed Looker-to-Omni workflow. A high score is not proof of query-result parity, access-filter equivalence, PDT behavior, or visual fidelity; those still require validation and human branch review.
